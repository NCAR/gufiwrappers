grprt¶

Generate report from raw data

usage: grprt [-h] [--gufitmp-dir= path-name]
             [--by-users | --by-projects | --by-subdirs-of= [Parent directory]]
             [--filter-by-unames= [User1,User2,..] [[User1,User2,..] ...]]
             [--filter-by-projects= [Project1,Project2,..]
             [[Project1,Project2,..] ...]]
             [--nthreads number-of-cores or processes]
             [--nsbins number-of-histogram bins [8]]
             treename

Positional Arguments¶

treename

Absolute path of the filesystem tree located in either in glade, campaign or HPSS

Named Arguments¶

--gufitmp-dir=

Absolute path name to store the GUFI query output default: /gpfs/fs1/scratch/sghosh/gufi_tmp

Default: “/gpfs/fs1/scratch/sghosh/gufi_tmp”

--by-users

report by user-name / user-ids (if mapping not found)

Default: False

--by-projects

report by project-name / project-id (if mapping not found) HPSS only

Default: False

--by-subdirs-of=

report by subdirectories of this parent directory

--filter-by-unames=

Report only for User1[,User2]..

--filter-by-projects=

Report only for Project1[,Project2]..

--nthreads

Number of cores / threads to run

Default: 1

--nsbins

Number of write / read stat histogram bins in report

Default: 8
